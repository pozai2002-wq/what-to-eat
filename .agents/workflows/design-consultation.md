# /design-consultation — Design System Creation

You are a design partner building a complete design system from scratch.

## Phase 1: Research
- Understand the product and its users
- Research competitor/adjacent product designs (use web search)
- Identify design patterns in the space
- Note what works and what's generic/boring

## Phase 2: Design System Foundation
Create DESIGN.md with:

### Colors
- Primary, secondary, accent palette (with hex values)
- Semantic colors (success, warning, error, info)
- Dark mode variants
- Background and surface colors

### Typography
- Font families (with Google Fonts links)
- Scale (headings h1-h6, body, caption, overline)
- Line heights and letter spacing

### Spacing
- Base unit and scale (4px, 8px, 12px, 16px, 24px, 32px, 48px, 64px)
- Component-level spacing rules

### Components
- Buttons (primary, secondary, ghost, destructive)
- Inputs (text, select, checkbox, radio, toggle)
- Cards, modals, tooltips
- Navigation patterns
- Loading states, empty states, error states

### Animations
- Transition timing (ease, duration)
- Micro-interactions (hover, focus, active)
- Page transitions
- Loading animations

## Phase 3: Visual Sketches
Generate wireframe HTML files for key screens:
- Self-contained, no external dependencies
- Intentionally rough aesthetic (system fonts, thin borders)
- Realistic placeholder content
- Show 1-3 screens/states

## Phase 4: Iterate
Present to user. Get feedback. Refine. Repeat until approved.

## Output
- DESIGN.md in repo root
- CSS custom properties / design tokens
- Example component implementations
